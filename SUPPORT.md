# Support

## Integration questions

Use [GitHub Discussions](https://github.com/GGULBAE/react-native-image-compression-kit/discussions)
for installation, API usage, capability fallback, and architecture questions.
Include the package version, React Native version, platform, OS/API level,
architecture mode, and a minimal code sample.

## Reproducible bugs

Use the appropriate [issue form](https://github.com/GGULBAE/react-native-image-compression-kit/issues/new/choose)
after checking existing issues. Build and compatibility reports should include
the requested environment matrix and relevant logs with credentials, paths,
and private image data removed.

## Security

Do not report vulnerabilities in Discussions or public issues. Follow the
private process in [SECURITY.md](SECURITY.md).

## Scope and response expectations

The project is maintained on a best-effort basis. Security reports are targeted
for acknowledgement within seven days. Bugs with a complete reproducer and
regressions in a declared compatibility lane receive priority. Feature requests
may remain open until they fit the public API and cross-platform capability
model.

The maintainer cannot provide private application debugging, guarantee a
resolution date, recover deleted output cache files, or support undocumented
codec behavior.
